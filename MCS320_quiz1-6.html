Quiz 1#

Consider \(\displaystyle r = \frac{1 + \sqrt{5}}{2}\).

  1. Compute a rational approximation for \(r\), accurate to 2 decimal places.

  2. Verify that the accuracy of your rational approximation is 2 decimal places.

    Write an explanation for your verification.

  3. Compute the sequence of the first ten consecutive rational approximations for \(r\), accurate respectively with 1, 2, up to 10 decimal places (10 included).

answer to question 1

r = (1 + sqrt(5))/2
show(r)
\(\displaystyle \frac{1}{2} \, \sqrt{5} + \frac{1}{2}\)
a = QQ(r.n(digits=2))
show(a)
\(\displaystyle \frac{55}{34}\)

answer to question 2

err = abs(r - a)/abs(r)
err.n(digits=2)

0.00

err.n(digits=3)

0.000226

We computed in err the relative error of the approximation a of the number r. Evaluated in with 2 decimal places of accuracy, we see that the error is 0.00. Alternatively, evaluate the error with 3 decimal places, we see that the error is \(2 \times 10^{-4}\). Thus, the accuracy of the approximation is (at least) 2 decimal places.

Quiz 3 MCS 320 Monday 29 June 2025

Question 1

Let \(\displaystyle \frac{1}{n} \sum_{k=1}^n \ln\left( 1 + \frac{k}{n} \right) \approx \int_0^1 \ln(1 + x) dx\)

where s = lambda n: float(sum([ln(1 + k/n) for k in range(1, n+1)]))/n

is a function which defines the approximation for the integral.

  1. Time the execution of s for \(n = 10000\). Explain why s is inefficient.

  2. Apply vectorization to improve the efficiency. Verify the correctness.

    Time the execution of the vectorized function for \(n = 10000\), compare with timings of s.

answer to question 1

s = lambda n: float(sum([ln(1 + k/n) for k in range(1, n+1)]))/n

timeit("s(10000)")

5 loops, best of 3: 691 ms per loop

The Python code is inefficient because all calculations are evaluated with exact rational arithmetic and only at the very end is a floating-point approximation computed.

def vs(n):
    """
    Vectorized version of s.
    """
    from numpy import log, sum, arange
    x = arange(0, n+1)/n
    return sum(log(1+x))/n

Let us first verify the correctness

s1 = s(10000)
s1

0.38632901806224984

s2 = vs(10000)
s2

0.38632901806225195

abs(s1-s2)

2.1094237467877974e-15

The error is about as small as the machine precision.

timeit("vs(10000)")

625 loops, best of 3: 54.8 Î¼s per loop

We observe a significant drop in execution time, from milliseconds to microseconds.

Question 2

Let \(p = x^3 + 2 x + 3\) be a polynomial with coefficients in the integer number field modulo 17.
Is \(p\) irreducible over \({\mathbb Z}_{17}\)?

Extend the coefficient field with a formal root of \(p\) so that, over this extended field, \(p\) factors into a product of linear polynomials.

answer to question 2

x = polygen(GF(17))
p = x^3 + 2*x + 3
p

x^3 + 2*x + 3

p.is_irreducible()

False

We see that \(p\) is reducible, let us then compute the factors.

f = factor(p)
f

(x + 1) * (x^2 + 16*x + 3)

To extend GF(17) with a root of a quadratic factor, we list the factors.

L = list(f)
L

[(x + 1, 1), (x^2 + 16*x + 3, 1)]

f1 = L[0][0]
f2 = L[1][0]
(f1, f2)

(x + 1, x^2 + 16*x + 3)

Now we can extend GF(17) with a root of the quadratic factor.

E.<a> = GF(17).extension(f2)
E

Finite Field in a of size 17^2

q = E[x](p)
factor(q)

(x + 1) * (x + a + 16) * (x + 16*a)

After casting \(p\) in to the extended number field, we see the three linear factors.

Question 2

Write an efficient recursive function to evaluate \(a(n) = a(n-1) + a(n-2) + a(n-3)\), with \(a(0) = 0\), \(a(1) = 1\), \(a(2) = 3\).

Make sure your function can compute \(a(100)\).

answer to question 2

reset()

def a(n, D={}):
    """
    Memoized version of the function a(n).
    """
    if n in D:
        return D[n]
    else:
        if n == 0:
            result = 0
        elif n == 1:
            result = 1
        elif n == 2:
            result = 3
        else:
            result = a(n-1) + a(n-2) + a(n-3)
        D[n] = result
        return result

Question 2

Consider the polynomial system

\[\begin{split} \left\{ \begin{array}{rcl} x^2 - 2 x y + 3 y^3 - 3 & = & 0 \\ x y + x - 4 & = & 0. \end{array} \right. \end{split}\]

  1. Bring the system into triangular form via a Groebner basis.

  2. Use the triangular form to justify the number of solutions.

  3. Solve the system.

answer to question 2

reset()

P.<x,y> = PolynomialRing(QQ, order='lex')

f = [x^2 - 2*x + 3*y^3 - 3, x*y + x - 4]
show(f)
\(\displaystyle \left[x^{2} - 2 x + 3 y^{3} - 3, x y + x - 4\right]\)
J = Ideal(f)
J

Ideal (x^2 - 2*x + 3*y^3 - 3, x*y + x - 4) of Multivariate Polynomial Ring in x, y over Rational Field

g = J.groebner_basis()
show(g)
\(\displaystyle \left[x + \frac{3}{4} y^{4} + \frac{3}{4} y^{3} - \frac{3}{4} y - \frac{11}{4}, y^{5} + 2 y^{4} + y^{3} - y^{2} - \frac{14}{3} y + \frac{5}{3}\right]\)

From the last two questions, we observe

  1. The last equation is a fifth degree polynomial only in \(y\), and

  2. The second to last equation allows to express \(x\) in function of \(y\).

So, for each of the five roots of the last equation, we have one corresponding \(x\) value. Therefore, we expect five solutions of the system. Let us solve the system.

sols = J.variety(QQbar)
sols

 {y: -1.961333621253331?, x: -4.160886409844931?},

 {y: -0.6931308912452093? - 1.401519354728868?*I, x: 0.5963182261314966? + 2.723478876359302?*I},
 {y: -0.6931308912452093? + 1.401519354728868?*I, x: 0.5963182261314966? - 2.723478876359302?*I}]

Indeed, we have five solutions: one is (2, 1), two are real, and two are complex conjugated.
